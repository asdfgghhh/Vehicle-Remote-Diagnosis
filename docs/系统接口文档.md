# 车辆远程诊断系统 — 系统接口文档

## 1. 通用约定

| 项目 | 说明 |
|------|------|
| 基础 URL | `http://{host}:8080/api` |
| 认证方式 | `Authorization: Bearer {token}`（登录/注册除外） |
| Content-Type | `application/json`（文件上传为 `multipart/form-data`） |
| 时间格式 | `yyyy-MM-dd HH:mm:ss` |
| 分页参数 | `current`（页码，默认 1）、`size`（每页条数，默认 10） |

### 1.1 统一响应结构

```json
{
  "code": 200,
  "message": "success",
  "data": "<业务数据>"
}
```

### 1.2 错误码

| code | 说明 |
|------|------|
| 200 | 成功 |
| 401 | 未认证 / 用户名密码错误 |
| 403 | 账号被禁用 |
| 500 | 业务异常（message 含具体原因） |

网关层 HTTP 401：请求未携带 `Authorization: Bearer ` 头（白名单接口除外）。

### 1.3 网关路由映射

对外路径 `/api/{service-path}` 经 StripPrefix=1 后转发至各微服务 `/{service-path}`。

## 2. 认证服务（service-auth :8081）

### 2.1 用户登录

- **POST** `/auth/login`
- **认证**：不需要
- **Content-Type**：`application/json`

**请求体**：

```json
{
  "username": "admin",
  "password": "your_password"
}
```

**响应 data**：

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "username": "admin",
  "userId": 1,
  "expiresIn": 86400000
}
```

**错误响应**：

| code | message |
|------|---------|
| 401 | 用户名或密码错误 |
| 403 | 账号已被禁用 |

---

### 2.2 用户注册

- **POST** `/auth/register`
- **认证**：不需要

**请求体**：

```json
{
  "username": "newuser",
  "password": "password123",
  "email": "user@example.com",
  "phone": "13800138000",
  "realName": "张三"
}
```

**响应 data**：`"注册成功"`（字符串）

---

### 2.3 校验 Token

- **GET** `/auth/validate`
- **请求头**：`Authorization: Bearer {token}`
- **响应 data**：`true` 或 `false`

---

### 2.4 用户管理

#### 2.4.1 用户分页

- **GET** `/auth/user/page`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| current | int | 否 | 页码，默认 1 |
| size | int | 否 | 每页条数，默认 10 |
| keyword | string | 否 | 用户名/姓名模糊搜索 |

**响应 data**：MyBatis-Plus `Page<UserVO>`（records, total, current, size）

#### 2.4.2 用户详情

- **GET** `/auth/user/{id}`
- **响应 data**：`UserVO`（含角色列表）

#### 2.4.3 创建用户

- **POST** `/auth/user`

**请求体（UserManageRequest）**：

```json
{
  "username": "operator1",
  "password": "password123",
  "email": "op@example.com",
  "phone": "13800138001",
  "realName": "操作员一",
  "status": 1,
  "roleIds": [2]
}
```

#### 2.4.4 更新用户

- **PUT** `/auth/user/{id}`
- **请求体**：同 UserManageRequest（password 可选）

#### 2.4.5 删除用户

- **DELETE** `/auth/user/{id}`
- **说明**：逻辑删除

#### 2.4.6 分配角色

- **PUT** `/auth/user/{id}/roles`

**请求体**：

```json
[1, 2]
```

### 4.2 车端上传（service-access :8086）

> 网关路径：`/api/ecu-log/vehicle/*` → service-access

#### 4.2.1 初始化分片上传

- **POST** `/ecu-log/vehicle/init`
- **Content-Type**：`application/x-www-form-urlencoded` 或 Query 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| vin | string | 是 | 车辆 VIN |
| ecuType | string | 是 | ECU 类型 |
| fileName | string | 是 | 文件名 |
| fileSize | long | 是 | 文件总大小（字节） |
| fileMd5 | string | 否 | 文件 MD5 |
| logStartTime | datetime | 是 | 日志起始时间 |
| logEndTime | datetime | 是 | 日志结束时间 |

**响应 data**：`uploadId`（字符串）

#### 4.2.2 上传分片

- **POST** `/ecu-log/vehicle/chunk`
- **Content-Type**：`multipart/form-data`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| uploadId | string | 是 | 初始化返回的上传 ID |
| chunkNumber | int | 是 | 分片序号（从 1 开始） |
| chunkSize | long | 否 | 分片大小 |
| file | file | 是 | 分片二进制数据 |

#### 4.2.3 完成分片上传

- **POST** `/ecu-log/vehicle/complete`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| uploadId | string | 是 | 上传 ID |

**响应 data**：`EcuLogRecord`

#### 4.2.4 整包直传

- **POST** `/ecu-log/vehicle/report`
- **Content-Type**：`multipart/form-data`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | file | 是 | 日志文件 |
| vin | string | 是 | 车辆 VIN |
| ecuType | string | 是 | ECU 类型 |
| logStartTime | datetime | 是 | 日志起始时间 |
| logEndTime | datetime | 是 | 日志结束时间 |
| fileMd5 | string | 否 | 文件 MD5 |

**分片上传流程**：`init` → `chunk`（循环）→ `complete`

### 6.2 车端上报（service-access :8086）

> 网关路径：`/api/signal/vehicle/*` → service-access

#### 6.2.1 HTTP 信号上报

- **POST** `/signal/vehicle/receive`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| vin | string | 是 | Query 参数，车辆 VIN |

- **请求体**：JSON 字符串（原始 payload，转发至 Kafka `vehicle-signals`）

**推荐 Kafka 消息格式**：

```json
{
  "vin": "LSVAG4189ES123456",
  "vehicleId": 1,
  "signals": [
    {
      "name": "VehicleSpeed",
      "value": "65.5",
      "unit": "km/h",
      "timestamp": 1704067200000,
      "messageName": "VehicleData",
      "messageId": 1024
    }
  ]
}
```

#### 6.2.2 MQTT 信号上报

- **协议**：MQTT
- **Broker**：`tcp://localhost:1883`
- **主题**：`vehicle/signal/+`（`+` 为 VIN 通配）
- **QoS**：1
- **处理**：`MqttSignalReceiver` 接收后发布至 Kafka `vehicle-signals`

## 9. 调用示例

### 9.1 登录并访问受保护接口

```bash
# 1. 登录
curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"your_password"}'

# 2. 携带 Token 查询车辆列表
curl http://localhost:8080/api/vehicle/page?current=1&size=10 \
  -H "Authorization: Bearer {token}"
```

### 9.2 车端整包上传日志

```bash
curl -X POST http://localhost:8080/api/ecu-log/vehicle/report \
  -H "Authorization: Bearer {token}" \
  -F "file=@/path/to/log.bin" \
  -F "vin=LSVAG4189ES123456" \
  -F "ecuType=EMS" \
  -F "logStartTime=2024-01-15 08:00:00" \
  -F "logEndTime=2024-01-15 09:00:00"
```

### 9.3 查询信号时间轴

```bash
curl "http://localhost:8080/api/signal/timeline/1?startTime=2024-01-15%2000:00:00&endTime=2024-01-15%2023:59:59" \
  -H "Authorization: Bearer {token}"
```
